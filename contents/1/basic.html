1.1 What Is Statistical Learning?

Statistical learning studies how to use data to understand relationships and make predictions. In most applications, we observe a response variable \(Y\) and a set of predictors \(X = (X_1, \dots, X_p)\), and we want to learn how \(Y\) depends on \(X\).

A common abstract representation is

\[ Y = f(X) + \varepsilon, \]

where \(f\) is the systematic part of the relationship and \(\varepsilon\) is random noise.

The main goals are usually:

  • prediction: accurately predict the response for new observations;
  • inference or explanation: understand how the predictors are related to the response.

1.2 Supervised and Unsupervised Learning

In supervised learning, we observe both predictors and a response. The goal is to predict or explain the response.

Examples include:

  • predicting house prices from size, location, and age;
  • predicting whether an email is spam from its content;
  • predicting a patient’s risk from clinical measurements.

In unsupervised learning, we observe predictors but no response. The goal is to discover structure in the data.

Examples include:

  • clustering similar observations;
  • reducing dimension with principal component analysis;
  • finding latent groups or patterns.

1.3 Regression and Classification

Within supervised learning, two major tasks are:

  • regression, where the response is quantitative;
  • classification, where the response is categorical.

For example:

  • predicting salary is a regression problem;
  • predicting whether a tumor is benign or malignant is a classification problem.

1.4 Prediction Versus Inference

Prediction and inference are related, but they are not the same.

If the goal is prediction, we care mainly about how well the model performs on new data. A more complex model may be acceptable if it improves predictive accuracy.

If the goal is inference, we care about understanding the role of each predictor. In this case, interpretability is often more important than pure predictive performance.

NoteMain Point

The best predictive model is not always the best explanatory model. Statistical learning often requires choosing between interpretability and flexibility.

1.5 Parametric and Nonparametric Methods

A parametric method assumes a specific functional form for \(f\). For example, linear regression assumes that

\[ f(X) = \beta_0 + \beta_1 X_1 + \cdots + \beta_p X_p. \]

This assumption simplifies estimation and interpretation, but it may be too restrictive.

A nonparametric method does not assume a fixed functional form. It is more flexible, but it usually requires more data and may be harder to interpret.

Thus:

  • parametric models are simpler and easier to interpret;
  • nonparametric models are more flexible but may have higher variance.

1.6 Training Error and Test Error

When we fit a model, we can measure how well it matches the observed data. This is the training error.

However, what really matters is performance on new observations. This is the test error.

A model can have very small training error but still perform poorly on new data. This phenomenon is called overfitting.

  • Underfitting occurs when the model is too simple to capture the signal.
  • Overfitting occurs when the model is too flexible and starts fitting noise.

Good statistical learning aims to minimize test error, not merely training error.

1.7 Bias-Variance Tradeoff

A central idea in statistical learning is the bias-variance tradeoff.

  • A simple model tends to have higher bias but lower variance.
  • A flexible model tends to have lower bias but higher variance.

As model flexibility increases:

  • training error usually decreases;
  • test error often decreases at first, then increases once overfitting begins.

The goal is to find a level of complexity that balances bias and variance well.

ImportantWhy This Matters

Many modern methods can be understood as ways to manage the bias-variance tradeoff. Regularization, cross-validation, and model comparison are all tools for this purpose.

1.8 A Typical Statistical Learning Workflow

A practical workflow often looks like this:

  1. Define the prediction or explanation goal.
  2. Explore and preprocess the data.
  3. Choose one or more candidate models.
  4. Estimate model performance using validation or cross-validation.
  5. Compare models and select an appropriate final model.
  6. Interpret results and communicate conclusions.

This course will revisit this workflow repeatedly with different methods.

1.9 Looking Ahead

The early chapters focus on linear regression and its extensions. Later chapters introduce more flexible models, resampling methods, regularization, and practical machine learning workflow.

The broad progression is:

  1. start with simple interpretable models;
  2. allow more flexible structure when needed;
  3. evaluate models using principled estimates of test error;
  4. improve prediction through regularization and modern methods.
