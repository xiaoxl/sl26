When the number of predictors is large or when predictors are highly correlated, directly fitting a regression model using the original predictors may lead to unstable estimates and poor predictive performance.

One approach to address this problem is dimension reduction. Instead of using the original predictors directly, we first construct a smaller number of derived variables (called components) and then perform regression using these components.

Two commonly used methods are

  • Principal Component Regression (PCR)
  • Partial Least Squares (PLS)

Both methods construct new variables that are linear combinations of the original predictors, but they differ in how those combinations are chosen.

3.1 PCR

PCR first applies principal component analysis (PCA) to the predictor matrix \(X\), and then uses the leading principal components as predictors in a regression model.

The key idea is that most of the variation in the predictors often lies in a lower-dimensional subspace.

Definition 3.1 (Principal Component Regression) Let \(X\) be the \(n \times p\) predictor matrix.

The principal components are defined as

\[ Z_k = X v_k, \]

where \(v_k\) is the \(k\)-th eigenvector of the Gram matrix \(X^\top X\).

Principal component regression fits a linear model using the first \(m\) components:

\[ y = \theta_0 + \theta_1 Z_1 + \theta_2 Z_2 + \cdots + \theta_m Z_m + \varepsilon. \]

Here

  • \(Z_1,\ldots,Z_m\) are the first \(m\) principal components
  • \(m < p\) is chosen by cross-validation

Thus PCR replaces the original predictors with a smaller set of orthogonal components.

3.1.1 PCA review

Principal component analysis finds directions in predictor space that maximize variance.

Let

\[ X = UDV^\top \]

be the singular value decomposition of \(X\).

Then

  • the columns of \(V\) are the principal directions
  • the principal components are

\[ Z = XV \]

The first principal component captures the largest variance in \(X\), the second captures the next largest variance orthogonal to the first, and so on.

NoteKey feature of PCR

PCR constructs components using only the predictor matrix \(X\), without considering the response \(y\).

As a result, the directions that explain the most variation in \(X\) are not necessarily the directions that are most predictive of \(y\).

4 Principal Component Analysis (PCA) Summary

4.1 1. Data Setup

Given a data matrix \(X \in \mathbb{R}^{n \times p}\) (\(n\) observations, \(p\) variables), we assume \(X\) is centered (each column has mean zero).

4.2 2. PCA via Singular Value Decomposition (SVD)

The SVD decomposes \(X\) as: \[X = UDV^\top\]

Component Matrix Dimension Interpretation
Directions \(V\) \(p \times r\) The weights (loadings) that define the new axes.
Scores \(Z = UD\) \(n \times r\) The coordinates of the data in the new PC space.
Magnitudes \(D\) \(r \times r\) Diagonal matrix of singular values \(d_k\).
Normalized \(U\) \(n \times r\) The directions of the scores (unit length).

4.3 3. Key Mathematical Properties

4.3.1 Principal Component Directions (\(v_k\))

The columns of \(V\) are orthonormal vectors that sequentially maximize variance: \[v_k = \arg\max_{\|v\|=1} \text{Var}(Xv)\] (subject to \(v_k \perp v_1, \dots, v_{k-1}\))

4.3.2 Principal Component Scores (\(z_i\))

The scores represent the original data projected onto the PC directions. For any observation \(x_i\): \[x_i = \sum_{k=1}^{r} z_{ik} v_k\]

4.4 4. Variance Explained

The importance of each component is determined by its singular value \(d_k\):

  • Variance of \(k\)-th component: \(\lambda_k = \frac{d_k^2}{n}\)
  • Proportion of Variance Explained (PVE): \[\text{PVE}_k = \frac{d_k^2}{\sum_{j=1}^{r} d_j^2}\]

4.5 5. Dimensionality Reduction

PCA allows for the best rank-\(m\) approximation (\(m < r\)) of the data by retaining only the first \(m\) columns of \(U, D,\) and \(V\): \[X \approx U_m D_m V_m^\top\] This reconstruction minimizes the squared reconstruction error, making it the optimal linear projection for data compression.

5 Principal Component Analysis

5.1 Data Setup

Suppose we observe a dataset with

  • \(n\) observations
  • \(p\) variables

Let the data matrix be

\[ X = \mqty[x_1^\top \\ x_2^\top \\ \vdots \\ x_n^\top ]=\mqty[X_1,X_2,\ldots,X_p]=\mqty[x_{ij}]_{1\leq i\leq n, 1\leq j\leq p} \in \mathbb{R}^{n\times p} \]

where

  • \(x_i \in \mathbb{R}^p\) is the \(i\)-th observation
  • \(X_j\in\mathbb{R}^n\) is the \(j\)-th variable

Throughout this section we assume the variables are centered, so each column of \(X\) has mean zero:

\[ \sum_{i=1}^{n} x_{ij} = 0,\quad \text{ for each }j. \]

Note

Centering the variables simplifies the PCA formulation and ensures that the first principal component corresponds to the direction of maximal variance.

6 PCA via Singular Value Decomposition

The singular value decomposition (SVD) of the data matrix \(X\) is

\[ X = UDV^\top \]

where

\[ U \in \mathbb{R}^{n\times r}, \quad D \in \mathbb{R}^{r\times r}, \quad V \in \mathbb{R}^{p\times r} \]

and

  • \(r = \rank(X)\)
  • \(U^\top U = I_r\)
  • \(V^\top V = I_r\)
  • \(D = \operatorname{diag}(d_1,\dots,d_r)\)
  • \(d_1 \ge d_2 \ge \dots \ge d_r >0\)

Here - The columns of \(UD\) are the principal component vectors, which is the direction of where the most variance are explained. It is official called principal component scores. - The columns of \(U\) are the normalized principal component vectors, which gives the direction, but not magnitude. - The diagnoal of \(D\) gives the magnitudes of each principal compoennt vector. - The columns of \(V\) are the pricipal component directions. In other words, it is the weights to indicate how to use linear combinations of the orignal variables to make the principal component vectors. The math is \(XV=UD\) since columns of \(V\) are orthogonormal.

8 Principal Component Directions

Let

\[ V = [v_1, v_2, \dots, v_r] \]

where

\[ v_k \in \mathbb{R}^p. \]

The vectors (v_k) are the principal component directions.

They are orthonormal:

\[ v_k^\top v_j = \begin{cases} 1 & k=j \\ 0 & k\ne j \end{cases} \]

Each direction maximizes variance sequentially:

\[ v_k = \arg\max_{\norm{v}=1} \operatorname{Var}(Xv) \]

subject to orthogonality constraints with previous components.

9 Principal Component Scores

The principal component scores are obtained by projecting the data onto the principal component directions:

\[ Z = XV \]

with

\[ Z \in \mathbb{R}^{n\times r}. \]

Using the SVD \(X=UDV^\top\),

\[ Z = XV = UD. \]

Thus

Matrix Dimension
\(Z\) \(n\times r\)
\(U\) \(n\times r\)
\(D\) \(r\times r\)

Each row of \(Z\) corresponds to the principal component representation of one observation.

10 Matrix Reconstruction Form

Using the SVD we can write

\[ X = UDV^\top \]

or equivalently

\[ X = ZV^\top \]

where

\[ Z = UD. \]

Dimension check:

\[ (n\times r)(r\times p) = n\times p. \]

Thus the original data matrix is reconstructed from

  • scores (Z)
  • principal directions (V).

11 Vector Form (Observation Representation)

Let

\[ z_i^\top \]

be the (i)-th row of (Z).

Then the (i)-th observation satisfies

\[ x_i = z_i^\top V^\top \]

or equivalently

\[ x_i = \sum_{k=1}^{r} z_{ik} v_k. \]

where

  • (v_k ^p) is the (k)-th principal component direction
  • (z_{ik}) is the score of observation (i) on component (k).

Thus each observation is expressed as a linear combination of the principal component directions.

12 Rank-1 Decomposition

The SVD can also be written as a sum of rank-1 matrices:

\[ X = \sum_{k=1}^{r} d_k u_k v_k^\top \]

where

  • \(u_k \in \mathbb{R}^n\)
  • \(v_k \in \mathbb{R}^p\)

Each matrix

\[ d_k u_k v_k^\top \]

represents the contribution of the \(k\)-th principal component to the data matrix.

13 Variance Explained

The sample covariance matrix is

\[ S = \frac{1}{n}X^\top X. \]

Using SVD,

\[ X^\top X = V D^2 V^\top. \]

Thus the variance explained by component \(k\) is

\[ \lambda_k = \frac{d_k^2}{n}. \]

The proportion of variance explained (PVE) is

\[ \text{PVE}_k = \frac{d_k^2}{\sum_{j=1}^{r} d_j^2}. \]

14 Dimensionality Reduction

If we retain only the first \(m\) principal components,

\[ X \approx Z_m V_m^\top \]

where

\[ Z_m \in \mathbb{R}^{n\times m}, \quad V_m \in \mathbb{R}^{p\times m}. \]

This provides the best rank-\(m\) approximation to \(X\) in terms of squared reconstruction error.

14.0.1 Choosing the number of components

The number of components \(m\) is usually chosen using cross-validation.

Typical procedure:

  1. Compute the principal components.
  2. Fit regression models using \(m = 1,2,\ldots,p\) components.
  3. Compute validation error for each \(m\).
  4. Choose the value of \(m\) with the lowest validation error.

14.0.2 PCR in Python

scikit-learn provides PCA for computing principal components.

from sklearn.decomposition import PCA
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LinearRegression

pcr_pipe = Pipeline(
    [
        ("scaler", StandardScaler()),
        ("pca", PCA(n_components=5)),
        ("model", LinearRegression())
    ]
)

pcr_pipe.fit(X, y)

Here the pipeline performs

standardization PCA regression using the principal components.

14.1 Partial Least Squares (PLS)

14.1.1 Idea

Principal component regression constructs components that explain the variation in the predictor matrix \(X\). However, these directions are chosen without considering the response variable \(y\).

Partial least squares (PLS) addresses this limitation by constructing components that are most strongly related to the response.

The key idea is to construct components that simultaneously

  • capture variation in the predictors, and
  • are strongly associated with the response.

Definition 14.1 (Partial Least Squares) Partial least squares constructs a sequence of components

\[ Z_k = X w_k \]

where the weight vector \(w_k\) is chosen to maximize the covariance between the component and the response:

\[ w_k = \arg\max_w \operatorname{Cov}(Xw, y)^2 \]

subject to the normalization constraint

\[ \|w\|_2 = 1. \]

After constructing the component \(Z_k\), both the predictor matrix and the response are deflated, and the procedure is repeated to construct additional components.

Thus PLS chooses directions in predictor space that are most predictive of the response.

14.1.2 Algorithm intuition

PLS constructs components sequentially.

At step \(k\):

  1. Find a direction \(w_k\) such that the component \(Z_k = X w_k\) has maximal covariance with \(y\).
  2. Regress \(y\) on the component \(Z_k\).
  3. Remove the variation explained by \(Z_k\) from both \(X\) and \(y\).
  4. Repeat the process on the residual matrices.

This produces components that are ordered by predictive importance.

14.1.3 Geometric interpretation

PLS can be understood geometrically as searching for directions in predictor space that align with the response.

  • PCR chooses directions with largest variance in \(X\).
  • PLS chooses directions with largest covariance between \(X\) and \(y\).

Therefore PLS tends to find directions that are more useful for prediction.

NoteComparison with PCR

PCR constructs components using only the predictor matrix \(X\).

PLS uses both \(X\) and \(y\) to determine the directions.

As a result

  • PCR may require many components before the response information is captured.
  • PLS often achieves good predictive performance with fewer components.

14.1.4 Choosing the number of components

Similar to PCR, the number of components is chosen using cross-validation.

Typical procedure:

  1. Fit PLS models with different numbers of components.
  2. Evaluate the validation error for each model.
  3. Choose the number of components that minimizes the validation error.

14.2 PLS in Python

scikit-learn provides the class PLSRegression for fitting partial least squares models.

from sklearn.cross_decomposition import PLSRegression
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler

pls_pipe = Pipeline(
    [
        ("scaler", StandardScaler()),
        ("model", PLSRegression(n_components=2))
    ]
)

pls_pipe.fit(X, y)

The argument n_components specifies the number of PLS components.

14.2.1 Selecting the number of components

We can choose the number of components using cross-validation.

from sklearn.model_selection import GridSearchCV

param_grid = {"model__n_components": list(range(1, X.shape[1] + 1))}

grid = GridSearchCV(pls_pipe, param_grid, cv=5, scoring="neg_mean_squared_error")

grid.fit(X, y)

best_components = grid.best_params_["model__n_components"]
best_components

14.2.2 PLS coefficient path

We can also examine how the regression coefficients change as the number of components increases.

import numpy as np
import matplotlib.pyplot as plt
from sklearn.cross_decomposition import PLSRegression
from sklearn.preprocessing import StandardScaler

X_scaled = StandardScaler().fit_transform(X)

coefs = []

for m in range(1, X.shape[1] + 1):
    pls = PLSRegression(n_components=m)
    pls.fit(X_scaled, y)
    coefs.append(pls.coef_.ravel())

coefs = np.array(coefs)

for j in range(coefs.shape[1]):
    plt.plot(range(1, X.shape[1] + 1), coefs[:, j], label=f"x{j + 1}")

plt.xlabel("Number of components")
plt.ylabel("Coefficient")
plt.title("PLS coefficient path")
plt.legend()
plt.show()

Summary

Partial least squares is a dimension reduction method designed for predictive modeling.

It constructs components that maximize covariance between predictors and response. These components are used as predictors in a regression model. The number of components is typically selected using cross-validation.

Compared with principal component regression:

PCR focuses on explaining predictor variation. PLS focuses on predicting the response.

As a result, PLS often requires fewer components to achieve strong predictive performance.
