6-Week Course Plan (Simplified)

This course introduces core statistical learning methods with an emphasis on intuition, interpretation, and practical implementation. Some advanced theoretical topics are omitted to keep the focus on applied understanding.

Primary references:

  • ISLR: An Introduction to Statistical Learning
  • ESL: The Elements of Statistical Learning
  • SMLR: Statistical Machine Learning with Math and R (Ruoqing Zhu)

Course Overview

This course is organized as an introduction to statistical learning from a statistical perspective. The focus is on three recurring ideas:

  1. Model flexibility
  2. Model evaluation
  3. Model improvement

Course Narrative

The course follows this progression:

Start with simple predictive models, make them more flexible, learn how to evaluate them, improve them, and finally apply them to real problems.

Learning Goals

By the end of the course, students should be able to:

  • identify common supervised learning problems,
  • distinguish regression and classification settings,
  • understand the role of model flexibility,
  • evaluate models using validation and resampling methods,
  • apply regularization and ensemble methods,
  • compare competing models using prediction performance,
  • implement statistical learning methods in software,
  • carry out a complete modeling workflow on real data.

Condensed Teaching Calendar

Unit Topic Main Goal
1 Foundations Introduce statistical learning and bias–variance
2 Linear Regression Establish a simple predictive baseline
3 Nonparametric Regression Introduce flexibility
4 Classification Shift from numeric to categorical prediction
5 Binary Classification Develop logistic and classification ideas
6 Generative Models Compare generative and discriminative methods
7 Resampling Learn to estimate test error
8 Regularization Control complexity and improve prediction
9 Ensemble Methods Combine models to improve performance
10 ML in Practice Execute a complete workflow

Final Summary

This course should leave students with a clear picture of statistical learning as a unified process:

  1. start with a modeling problem,
  2. choose an appropriate predictive method,
  3. evaluate it using principled tools,
  4. improve it when needed,
  5. apply it carefully in practice.

Unit 1: Foundations of Statistical Learning

Main Topics

  • What is statistical learning?
  • Supervised vs unsupervised learning
  • Parametric vs nonparametric methods
  • Prediction vs explanation
  • Bias–variance tradeoff

Teaching Goals

Students should understand that the main goal of statistical learning is good prediction on new data, not simply fitting the observed sample as closely as possible.

Suggested Emphasis

  • Introduce the overall modeling problem first.
  • Use simple real-world examples such as house prices, medical diagnosis, or spam filtering.
  • Make overfitting a central theme from the beginning.

Unit 2: Linear Regression as a Baseline

Main Topics

  • Linear regression as a prediction tool
  • Interpreting fitted relationships
  • Strengths and weaknesses of linear models
  • Linear models as a baseline for comparison

Teaching Goals

Students should see linear regression as the starting point of predictive modeling: simple, interpretable, and useful, but limited in flexibility.

Suggested Emphasis

  • Present linear regression mainly as a predictive model.
  • Keep inference light unless needed for context.
  • Highlight when linear models fail to capture nonlinear structure.

Unit 3: Nonparametric Regression

Main Topics

  • Flexible regression methods
  • K-nearest neighbors regression
  • Smoothers and local fitting ideas
  • Model flexibility and overfitting

Teaching Goals

Students should learn that more flexible models can better capture complicated patterns, but at the cost of increased variance. ## Suggested Emphasis - Connect this unit directly to bias–variance tradeoff. - Compare flexible and inflexible models visually. - Use plots to show underfitting and overfitting.

Unit 4: Classification

Main Topics

  • Classification as a prediction problem
  • Logistic regression
  • Decision boundaries
  • K-nearest neighbors classification

Teaching Goals

Students should understand that classification uses the same modeling logic as regression, but with categorical outcomes. ## Suggested Emphasis - Stress the difference between predicting a class and predicting a numeric response. - Introduce class probabilities early. - Use confusion matrices and classification error as evaluation tools.

Unit 5: Binary Classification

Main Topics

  • Logistic regression in more detail
  • Interpreting probabilities
  • Thresholds and classification decisions
  • Comparing classifiers

Teaching Goals

Students should learn how binary classification models convert predictors into class probabilities and final decisions. ## Suggested Emphasis - Distinguish between probability estimation and hard classification. - Discuss how different thresholds change predictions. - Use practical examples such as disease detection or spam filtering.

Unit 6: Generative Models

Main Topics

  • LDA
  • QDA
  • Naive Bayes
  • Discriminative vs generative modeling

Teaching Goals

Students should understand the idea of modeling class distributions directly and then using Bayes’ rule for prediction. ## Suggested Emphasis - Compare logistic regression with LDA and QDA conceptually. - Emphasize assumptions and when they matter. - Use side-by-side model comparisons.

Unit 7: Resampling Methods

Main Topics

  • Validation set approach
  • Cross-validation
  • Bootstrap
  • Estimating test error

Teaching Goals

This unit is the conceptual center of the course. Students must understand that training error is not enough, and that model performance must be evaluated using principled estimates of test error. ## Suggested Emphasis - Make cross-validation a central practical tool. - Show why naive comparison by training error is misleading. - Use repeated examples across multiple methods.

Unit 8: Regularization

Main Topics

  • Ridge regression
  • Lasso
  • Elastic net
  • Shrinkage and complexity control

Teaching Goals

Students should understand how regularization controls complexity and improves prediction without exhaustively searching across models. ## Suggested Emphasis - Present regularization as a modern answer to overfitting in linear models. - Compare subset selection and shrinkage conceptually. - Use coefficient path plots when possible.

Unit 9: Ensemble Methods

Main Topics

  • Decision trees
  • Bagging
  • Random forests
  • Boosting

Teaching Goals

Students should learn that combining many unstable or weak learners can create a stronger predictive model. ## Suggested Emphasis - Start with a single tree and its limitations. - Then motivate bagging as variance reduction. - Finally introduce random forests and boosting as improved ensemble ideas.

Unit 10: Machine Learning in Practice

Main Topics

  • End-to-end modeling workflow
  • Data preprocessing
  • Model tuning
  • Model comparison
  • Practical considerations on real datasets

Teaching Goals

Students should be able to complete a full predictive analysis from data setup through final model comparison and interpretation. ## Suggested Emphasis - Organize this unit around a full case study. - Show how preprocessing, resampling, tuning, and comparison fit together. - Emphasize reproducible workflow and communication.

Exams and Major Checkpoints

Midterm I

Suggested coverage:

  • foundations of statistical learning,
  • linear regression,
  • nonparametric regression,
  • early classification concepts.

Midterm II

Suggested coverage:

  • classification,
  • generative models,
  • resampling,
  • regularization.

Final Assessment or Project

Suggested format:

  • a real dataset,
  • several competing methods,
  • model comparison using validation or cross-validation,
  • written interpretation of results.

Weekly Teaching Rhythm

A consistent teaching rhythm can help students stay oriented. ## Lecture 1 - concept and intuition, - visual explanation, - strengths and limitations of the method.

Lecture 2

  • implementation,
  • model fitting,
  • comparison with earlier methods,
  • short applied example.

Core Themes to Revisit Throughout the Course

Theme 1: Model Flexibility

Students should repeatedly ask: > How flexible is this model, and what does that flexibility buy us? ## Theme 2: Generalization Students should repeatedly ask: > How do we know whether this model will work on new data? ## Theme 3: Comparison Students should repeatedly ask: > Is this method better than what we already had? # Recommended Course Message to Students A useful recurring message for students is: > Statistical learning is about building models that perform well on new data, understanding the tradeoff between simplicity and flexibility, and choosing methods based on predictive performance. # Condensed Teaching Calendar

Gemini

This course plan integrates the theoretical depth of The Elements of Statistical Learning (ESL) and the computational focus of Ruoqing Zhu’s SMLR and STAT 432 with the intuitive, application-based approach of An Introduction to Statistical Learning (ISLR). The curriculum is designed for a 15-week semester, prioritizing numerical optimization (as emphasized in SMLR) alongside standard statistical learning theory.

Phase 1: Foundations & Optimization (Weeks 1–3)

Focus: Setting the stage with R, Linear Algebra, and the “How” of algorithm fitting.

  • Week 1: Introduction & The Learning Framework
    • What is Statistical Learning? (ISLR Ch. 2; ESL Ch. 2).
    • Supervised vs. Unsupervised Learning.
    • The Bias-Variance Trade-off and MSE (SMLR Ch. 12.3).
  • Week 2: Computational Toolkit
    • R & RMarkdown: Environment setup and reproducible workflows (SMLR Ch. 1-2).
    • Linear Algebra Refresher: Matrix inversion, rank updates, and block matrices (SMLR Ch. 4).
  • Week 3: Optimization Basics
    • First and Second-order properties (Gradient vs. Hessian).
    • Algorithms: Gradient Descent, Newton’s Method, and Coordinate Descent (SMLR Ch. 5).
    • Lab: Implementing Linear Regression using optim() from scratch. ### Phase 2: Linear Models & Regularization (Weeks 4–6) Focus: Moving beyond Ordinary Least Squares to handle high-dimensional data.
  • Week 4: Linear Regression & Model Selection
    • OLS properties and limitations.
    • Criteria: \(C_p\), AIC, BIC, and Adjusted \(R^2\) (SMLR Ch. 6; ISLR Ch. 6).
    • Search: Best subset vs. Stepwise methods.
  • Week 5: Penalized Regression (Ridge & Lasso)
    • Ridge Regression: Motivation via correlated variables and convexity (SMLR Ch. 7).
    • Lasso: The \(L_1\) penalty, sparsity, and the soft-thresholding operator (SMLR Ch. 8).
    • Elastic Net: Combining \(L_1\) and \(L_2\) penalties.
  • Week 6: Path Algorithms & Cross-Validation
    • The Lasso solution path and Path-wise Coordinate Descent (SMLR Ch. 8.4).
    • k-fold and Leave-one-out Cross-Validation (ISLR Ch. 5).
    • Lab: The glmnet package in R. ### Phase 3: Classification & Smoothing (Weeks 7–9) Focus: Categorical outcomes and non-linear boundaries.
  • Week 7: Linear Classification
    • Logistic Regression: Parameters, interpretation, and Newton-Raphson for MLE (SMLR Ch. 10).
    • Discriminant Analysis (LDA/QDA): Bayes’ Rule and decision boundaries (ISLR Ch. 4; SMLR Ch. 11).
  • Week 8: Non-parametric Smoothing & Splines
    • Moving beyond linearity: Polynomials and Piecewise functions.
    • Splines: Basis functions, Natural Cubic Splines, and Smoothing Splines (SMLR Ch. 9; ISLR Ch. 7).
  • Week 9: Local Learning & Kernels
    • K-Nearest Neighbors (KNN) for regression and classification (SMLR Ch. 12).
    • Kernel Density Estimation and Nadaraya-Watson (SMLR Ch. 13).
    • The “Curse of Dimensionality.” ### Phase 4: Support Vector Machines & Kernels (Weeks 10–11) Focus: The “Kernel Trick” and Maximal Margins.
  • Week 10: Support Vector Machines (SVM)
    • The Maximal Margin Classifier and Slack Variables (SMLR Ch. 17).
    • Duality: From Primal to Dual formulations (ESL Ch. 12).
  • Week 11: RKHS & Kernel Methods
    • Reproducing Kernel Hilbert Space (RKHS) basics (SMLR Ch. 15).
    • Kernel Ridge Regression and the Kernel Trick (SMLR Ch. 16). ### Phase 5: Trees, Ensembles & Unsupervised Learning (Weeks 12–15) Focus: Modern “Black Box” winners and discovery without labels.
  • Week 12: Tree-Based Methods
    • Decision Trees: Gini index vs. Entropy (ISLR Ch. 8).
    • Bagging and Random Forests (SMLR Ch. 19-20).
  • Week 13: Boosting
    • AdaBoost and the Stagewise Additive Model.
    • Gradient Boosting Machines (GBM) and XGBoost/LightGBM concepts (SMLR Ch. 22-23).
  • Week 14: Unsupervised Learning I (Clustering)
    • K-means and Hierarchical Clustering (SMLR Ch. 24-25).
    • Spectral Clustering and the Graph Laplacian (SMLR Ch. 28).

While scikit-learn is the industry standard for prediction, statsmodels is the primary library for statistical inference (\(p\)-values, \(R^2\), and \(t\)-tests) and matches the R lm() experience.

Feature R Code Python (statsmodels.formula.api)
Linear Regression fit <- lm(y ~ x1 + x2, data=df) fit = smf.ols("y ~ x1 + x2", data=df).fit()
Logistic Regression glm(y ~ x, data=df, family="binomial") smf.logit("y ~ x", data=df).fit()
Model Summary summary(fit) print(fit.summary())
ANOVA Table anova(fit) sm.stats.anova_lm(fit)
Predictions predict(fit, newdata) fit.predict(new_data)
Interaction Terms lm(y ~ x1 * x2) smf.ols("y ~ x1 * x2", data=df).fit()
Polynomials lm(y ~ poly(x, 2)) smf.ols("y ~ x + I(x**2)", data=df).fit()

2. Model Selection & Regularization

Python’s scikit-learn (sklearn) is the powerhouse for cross-validation and penalized regression.

Feature R Code (Package) Python Library Python Counterpart
Lasso / Ridge glmnet(x, y) sklearn LassoCV() or RidgeCV()
Stepwise (AIC/BIC) step(fit) sklearn SequentialFeatureSelector
Best Subset regsubsets() (leaps) mlxtend ExhaustiveFeatureSelector

3. Diagnostics & Residuals

In R, plot(fit) automatically generates four diagnostic plots. In Python, you access these via the .resid attribute or statsmodels.graphics.

Metric / Plot R Code Python Code
Residuals residuals(fit) fit.resid
Fitted Values fitted(fit) fit.fittedvalues
Q-Q Plot qqnorm(fit$resid) sm.qqplot(fit.resid, line='s')
Leverage Plot plot(fit, which=4) sm.graphics.influence_plot(fit)
Residual Plot plot(fit, which=1) sns.residplot(x=fit.fittedvalues, y=df['y'])
